# 1倍高频网格策略详细注释文档 (v6 修正版)

## 一、策略概述

- **策略类型**：双向对冲高频网格（现货等价，无杠杆）
- **适用交易所**：Bitfinex（0 手续费环境）
- **杠杆倍数**：1 倍（不借钱，不可能爆仓）
- **下单间距**：0.01%（极密网格）
- **止盈间距**：0.02%（每笔赚取入场间距的 2 倍）
- **网格层数**：单向 20 层，双向共 40 层
- **重置周期**：60 秒（1 分钟）

---

## 二、为什么选择 1 倍？

| 杠杆 | 爆仓线 | 实际风险 |
|:---|:---|:---|
| 30x | 3.3% | 极危险，几秒就可能爆仓 |
| 10x | 10% | 高危，一天内可能爆仓 |
| 5x | 20% | 中等，极端行情可能爆仓 |
| **1x** | **不可能爆仓** | **和持有现货完全等价，BTC 不归零就不会归零** |

1 倍的核心优势：**交易所不会强平你的仓位**。你的资产永远在，最差的情况就是全部被套住、暂时无法下新单。

## 三、全部参数详解

### 3.1 资金参数

| 参数名 | 值 | 含义 |
|:---|:---|:---|
| `INITIAL_CASH` | 1000.0 | 初始本金（USDT）。所有计算的起点。 |
| `CAPITAL_UTILIZATION` | 0.95 | 资金利用率。每次重置时只使用可用余额的 95%，预留 5% 作为缓冲。 |

### 3.2 网格参数

| 参数名 | 值 | 含义 |
|:---|:---|:---|
| `ENTRY_SPACING_PCT` | 0.01 | **下单间距（百分比）**。相邻两层挂单之间的价格差距为当前价格的 0.01%。 |
| `TP_SPACING_PCT` | 0.02 | **止盈间距（百分比）**。成交后，止盈单挂在入场价上方/下方 0.02% 处。 |
| `LEVELS_PER_SIDE` | 20 | 单向层数。向下挂 20 层买单，向上挂 20 层卖单，共 40 层。 |
| `RE_CENTER_US` | 60000000 | 重置周期（微秒）。60,000,000 微秒 = 60 秒 = 1 分钟。 |

### 3.3 间距与利润的数学关系

#### 下单间距 vs 止盈间距（核心设计）
```
下单间距 = 0.01%  -> 决定"多密集地布网"
止盈间距 = 0.02%  -> 决定"每笔赚多少"

为什么止盈 > 下单？
  如果止盈 = 下单（都是 0.01%），每笔利润极小。
  止盈设为 2 倍间距，意味着：
    - 价格跌 0.01% 买入
    - 价格需要涨 0.02% 才止盈（而不是涨回原价）
    - 每笔利润翻倍，但止盈概率略低
```

#### 不同 BTC 价格下的精确计算

| BTC 价格 | 下单间距(美元) | 止盈间距(美元) | 20层总跨度 | 每格名义 | 单笔利润 |
|:---|:---|:---|:---|:---|:---|
| $60,000 | $6.00 | $12.00 | $120 | $23.75 | $0.00475 |
| $80,000 | $8.00 | $16.00 | $160 | $23.75 | $0.00475 |
| $95,000 | $9.50 | $19.00 | $190 | $23.75 | $0.00475 |

```
计算过程（以 BTC=$95,000 为例）：
  可用余额 = 1000 x 0.95 = $950
  每格名义 = $950 / 40 = $23.75
  
  第1层买单：95,000 x (1 - 0.0001) = $94,990.50
  第1层卖单：95,000 x (1 + 0.0001) = $95,009.50
  第20层买单：95,000 x (1 - 20x0.0001) = $94,810.00
  第20层卖单：95,000 x (1 + 20x0.0001) = $95,190.00
  
  买单总跨度：95,000 - 94,810 = $190
  卖单总跨度：95,190 - 95,000 = $190
  完整跨度：$380
  
  单笔利润 = 名义 x 止盈百分比 = 23.75 x 0.0002 = $0.00475
```

## 四、资金管理逻辑（核心：为什么不会爆仓）

### 4.1 保证金模型

在 1 倍杠杆（现货等价）下，交易所的保证金规则如下：

```
总资金 = 初始本金 + 已实现盈亏
已冻结保证金 = 持仓占用 + 挂单占用
  持仓占用 = sum(每个持仓的 数量 x 入场价)
  挂单占用 = sum(每个挂单的 名义价值)
可用余额 = 总资金 - 已冻结保证金

规则：
  if 可用余额 >= 新订单名义价值:
    允许下单
  else:
    拒绝下单（余额不足）
```

### 4.2 资金冻结流转图

```
[可用余额 $1000]
       |
       |---> 挂单冻结 $950 (40个挂单 x $23.75)
       |     剩余可用 $50
       |
       |---> 买单成交：挂单冻结 -> 持仓冻结（金额不变）
       |     可用余额不变
       |
       |---> 止盈平仓：持仓冻结释放 + 利润入账
       |     可用余额增加 = 释放的保证金 + $0.00475 利润
       |
       |---> 下次重置：基于新的可用余额重新计算每格名义
```

### 4.3 极端情况分析

#### 场景：BTC 单边暴跌 30%
```
假设所有 20 层买单全部成交，价格继续下跌。
  20 个多单被套，冻结了约 $475 保证金。
  挂单被取消（重置时），释放挂单冻结。
  剩余可用余额 = $1000 + 已实现利润 - $475

会不会爆仓？
  不会。1x 杠杆下，即使 20 个多单全部浮亏 30%：
    浮亏 = 20 x $23.75 / $95,000 x 0.3 x $95,000 = 20 x $7.125 = $142.5
    账户净值 = $1000 - $142.5 = $857.5
    仍然有 $857.5 的净值，只是其中 $475 被持仓冻结。
    可用余额 = $857.5 - $475 = $382.5
    机器人仍然可以继续下单！

会不会停工？
  可能会暂时减少下单数量（因为可用余额减少了），
  但不会完全停工。
```

#### 场景：BTC 先跌 10% 再涨回来（V 型反转）
```
下跌阶段：
  部分买单成交（比如 10 层），被套住。
  可用余额减少，新一轮挂单数量减少。
  但空单网格在高位止盈，持续产生利润。

反弹阶段：
  被套的多单逐个止盈，释放保证金 + 赚取利润。
  可用余额恢复，回到满额挂单状态。
  
结果：整个 V 型过程中，机器人赚了"下跌时空单的利润 + 反弹时多单的利润"。
```

## 五、与买入持有的对比逻辑

| 策略 | 上涨行情 | 震荡行情 | 下跌行情 |
|:---|:---|:---|:---|
| **买入持有** | 全额盈利 | 无盈利 | 全额亏损 |
| **网格策略** | 部分盈利（空单被套） | **持续盈利（核心优势）** | 部分亏损（多单被套，但空单止盈抵消部分） |

网格策略的优势在于**震荡行情**：BTC 在一个区间内反复波动时，买入持有毫无收益，但网格每次波动都能赚取 0.02% 的差价。
